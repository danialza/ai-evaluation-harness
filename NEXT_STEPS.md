# Next Steps

Here is the simple path I would take next.

## 1. Keep the first version tiny

The current version is good for learning because it is small.

Do not rush into agents, vector databases, or dashboards yet.

## 2. Add a real model connector

Next, replace the fake model with one real option:

- OpenAI API
- Ollama local model
- Hugging Face local model

Pick only one first.

## 3. Save results to JSON

After each run, save:

- question
- expected answer
- model answer
- pass/fail
- final score

This makes it easier to show results on your website.

## 4. Add a tiny web page

For danielz.co.uk, turn this into a small project page:

- short explanation
- diagram
- code snippet
- screenshot of the terminal output
- GitHub repo link

## 5. Post on LinkedIn

Keep the LinkedIn post simple:

"I built a tiny AI Evaluation Harness from scratch to understand how models are tested. It asks questions, sends them to a model, grades the answers, and prints a score."

That is clear and honest, which is exactly what makes it good.
